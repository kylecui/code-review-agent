# v0.2.0 Release Notes — "天眼" Multi-Engine Fusion SAST

## Overview

v0.2.0 transforms Agent Review from a single-engine scanner into a **multi-engine fusion SAST platform** with three-tier engine architecture, 8 new collectors, universal SARIF support, and SaaS-ready per-user configuration.

## New Features

### Multi-Engine Architecture (L1/L2/L3)

- **L1 — Universal**: Semgrep, Gitleaks, Secrets (run on every scan)
- **L2 — Language-Specific**: SpotBugs (Java), golangci-lint (Go), Cppcheck (C/C++), ESLint Security (JS/TS), Roslyn (C#), Luacheck (Lua)
- **L3 — Deep Analysis**: CodeQL (baseline scans only, supports Java/Go/Python/C/C++/C#/JS/TS/Ruby)

### Engine Router

Automatic language detection from file extensions with intelligent engine selection per scan track:
- **Incremental (PR)**: L1 + L2 collectors based on detected languages
- **Baseline**: L1 + L2 + L3 (CodeQL) for comprehensive coverage

### 8 New Collectors

| Collector | Engine | Languages | Output Format |
|-----------|--------|-----------|---------------|
| GitleaksCollector | L1 | All | SARIF |
| SpotBugsCollector | L2 | Java | SARIF |
| GolangciLintCollector | L2 | Go | SARIF |
| CppcheckCollector | L2 | C/C++ | SARIF |
| EslintSecurityCollector | L2 | JS/TS | JSON |
| RoslynCollector | L2 | C# | JSON |
| LuacheckCollector | L2 | Lua | Plain text |
| CodeQLCollector | L3 | Multi | SARIF |

### SARIF v2.1.0 Adapter

Universal parser for SARIF output that extracts: rule metadata, CWE references, precision tags, code snippets, and partial fingerprints. Used by 5 of the 8 new collectors.

### Enhanced Fingerprinting

AST-aware `fingerprint_v2` generation that survives whitespace/formatting changes. The deduplicator now groups by `fingerprint_v2` first (when set), falling back to legacy `fingerprint`.

### Reachability Analysis

SCA+SAST correlation engine that demotes unreachable dependency findings by one severity level. Identifies call-site evidence by cross-referencing SCA findings against SAST findings in the same codebase.

### SaaS Multi-Tenancy

- **Per-user LLM API keys**: Each user stores their own OpenAI/Gemini/GitHub/Anthropic keys, encrypted at rest
- **Per-user repository configuration**: Users manage their own GitHub/GitLab/Bitbucket repo connections
- **Tenant isolation**: UserSettings and UserRepository models enforce strict per-user boundaries
- **3-layer settings resolution**: User overrides → Global AppConfig → Environment variables

### API Endpoints

- `GET/PUT/DELETE /api/user/settings/{key}` — Per-user encrypted settings management
- `GET/POST /api/user/repositories` — Per-user repository CRUD
- `GET/PUT/DELETE /api/user/repositories/{id}` — Individual repository management

## Configuration

All new collectors are **disabled by default** for backward compatibility. Enable them in your `.env` or via the admin settings API:

```env
GITLEAKS_ENABLED=true
SPOTBUGS_ENABLED=true
GOLANGCI_LINT_ENABLED=true
CPPCHECK_ENABLED=true
ESLINT_SECURITY_ENABLED=true
ROSLYN_ENABLED=true
LUACHECK_ENABLED=true
CODEQL_ENABLED=true
```

Engine tier configuration in `policies/default.policy.yaml`:

```yaml
engine_tiers:
  incremental:
    l1_enabled: true
    l2_enabled: true
    l3_enabled: false
  baseline:
    l1_enabled: true
    l2_enabled: true
    l3_enabled: true
```

## Database Migration

Run the new migration to add v0.2.0 tables and columns:

```bash
alembic upgrade head
```

This adds:
- `user_settings` table (per-user encrypted key-value store)
- `user_repositories` table (per-user repo connections)
- `fingerprint_v2` and `engine_tier` columns on `finding`
- `detected_languages` and `engine_selection` columns on `review_run`

## Docker

The updated Dockerfile includes Gitleaks, Cppcheck, and Luacheck. Optional tools (SpotBugs, golangci-lint, Roslyn, CodeQL, ESLint) should be installed separately based on your language needs.
